We will use React.memo on ChatBubble and DateSeparator components.
In a long list with `react-virtuoso`, `renderItem` often calls a stateless component like `ChatBubble`. While `react-virtuoso` unmounts and remounts components during scroll, a parent state change (like typing in the search bar or changing slider level if `ChatViewer` re-renders and causes a new array of filtered messages or search results) can cause all currently mounted rows to re-render. Since `ChatBubble` and `DateSeparator` are purely presentational based on props, `React.memo` will prevent re-rendering identical chat bubbles that are on screen.
